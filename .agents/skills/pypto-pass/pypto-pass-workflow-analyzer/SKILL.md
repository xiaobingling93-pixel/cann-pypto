# PyPTO Pass Workflow Analyzer Skill

## 概述

本技能用于分析 PyPTO pass 文档中的业务流介绍部分，帮助理解 pass 的执行流程和数据流转。

## 功能

### 基础业务流分析
- 解析 pass 文档中的业务流介绍章节
- 提取业务流的关键信息：
  - 业务流阶段划分
  - 各阶段的执行顺序
  - 数据流转路径
  - 关键决策点
  - 异常处理流程

### 视图类OP流程分析（新增）
- 识别文档中对视图类OP的特殊处理：
  - View 操作
  - Assemble 操作
  - Reshape 操作
- 按每个视图类OP角度描绘处理流程：
  - 该OP在Pass中的触发时机
  - 该OP对数据流的影响
  - 该OP与其他模块的交互
  - 该OP的特殊处理逻辑
  - 该OP的约束条件和限制
- 按照Pass模块执行顺序分析：
  - 严格遵循Pass中各模块的执行顺序
  - 在每个模块前后绘制OP状态变化
  - 标注关键转换点和数据变化

## 使用场景

当需要理解 PyPTO pass 的执行流程和业务逻辑时使用此技能，特别是：
- 需要了解视图类OP（View/Assemble/Reshape）在Pass中的处理流程
- 需要分析视图类OP对数据流的影响
- 需要理解视图类OP与其他模块的交互关系

### 调用方式示例

**基础业务流分析：**
- "分析Pass XXX业务流"
- "分析Pass的整体执行流程"
- "解析Pass XXX的数据流转"

**视图类OP流程分析：**
- "分析Pass View使用"
- "分析Pass Assemble的处理流程"
- "分析Pass Reshape是怎么被优化的"
- "分析Pass中View/Assemble/Reshape的完整处理流程"
- "从View角度分析Pass的数据流变化"

**批量分析：**
- "分析Pass文档路径下的所有视图类OP流程"
- "对指定文档集合进行视图类OP流程分析"

## 工作流程

### 基础业务流分析
1. 定位 pass 文档中的业务流介绍部分
2. 分析业务流的结构和阶段
3. 提取业务流的关键信息
4. 绘制业务流执行路径

### 视图类OP流程分析（新增）
1.输入处理：
   - 接收文件夹路径或一系列文档路径
   - 读取所有相关文档内容

2.视图类OP识别：
   - 搜索文档中包含 View/Assemble/Reshape 关键词的章节
   - 提取每个视图类OP的相关描述
   - 标记视图类OP在文档中的位置和上下文

3.Pass模块执行顺序识别：
   - 从文档中提取Pass的模块执行顺序
   - 确定每个模块的名称和职责
   - 建立模块间的先后关系

4.流程分析（针对每个视图类OP）：
   - 按照Pass模块执行顺序逐个分析
   - 对每个模块，识别该OP在模块前的状态
   - 分析该OP在模块中的处理逻辑
   - 描述该OP在模块后的状态变化
   - 使用流程图展示模块前后的状态对比

5.流程整合：
   - 按视图类OP类型（View/Assemble/Reshape）分类整理
   - 为每个OP绘制按模块顺序的完整流程图
   - 标注每个模块前后的关键变化
   - 不输出视图OP间的依赖关系
