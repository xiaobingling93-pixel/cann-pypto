## 致命优先级（9 个）

直接导致任务失败，必须修复。

### D1 - 文档缺失

- **优先级**：致命
- **层级**：实体级
- **根因归属**：文档
- **信号描述**：搜索某个 API/概念但找不到任何文档
- **典型表现**：Grep/Glob 搜索返回空结果，Read 找不到对应文件
- **示例**：搜索 `pypto.reshape` 的文档，但整个文档目录中没有任何相关内容

### D3 - 文档误导

- **优先级**：致命
- **层级**：实体级
- **根因归属**：文档
- **信号描述**：按文档操作失败，文档与实际行为不一致
- **典型表现**：读取了文档，按文档说明编写代码，但运行时报错或结果不符
- **示例**：文档说 `pypto.add` 支持广播，但实际调用时报 shape 不匹配

### D6 - 示例无效

- **优先级**：致命
- **层级**：实体级
- **根因归属**：文档
- **信号描述**：文档中的示例代码无法运行或报错
- **典型表现**：复制文档中的示例代码运行后报 ImportError、AttributeError 等
- **示例**：文档示例使用了已废弃的 API 签名

### A1 - API 不存在

- **优先级**：致命
- **层级**：实体级
- **根因归属**：框架
- **信号描述**：调用某个 API 但返回"未定义"或"不存在"
- **典型表现**：`AttributeError: module 'pypto' has no attribute 'xxx'`
- **示例**：尝试调用 `pypto.reshape` 但该 API 不存在

### A2 - API 行为异常

- **优先级**：致命
- **层级**：实体级
- **根因归属**：框架
- **信号描述**：API 返回结果与预期严重不符
- **典型表现**：API 执行成功但输出数据错误，或静默返回错误结果
- **示例**：`pypto.matmul` 返回的 tensor shape 与数学预期不一致

### E1 - 错误信息缺失

- **优先级**：致命
- **层级**：实体级
- **根因归属**：框架
- **信号描述**：操作失败但没有任何错误提示
- **典型表现**：程序静默失败、返回空结果、或挂起无响应
- **示例**：编译 pass 失败但没有任何输出

### E2 - 错误信息模糊

- **优先级**：致命
- **层级**：实体级
- **根因归属**：框架
- **信号描述**：错误信息太笼统，无法定位问题
- **典型表现**：`Error: internal error`、`Failed`、`Unknown error occurred`
- **示例**：codegen 失败只提示 "compilation failed" 没有具体原因

### E3 - 错误信息误导

- **优先级**：致命
- **层级**：实体级
- **根因归属**：框架
- **信号描述**：错误信息指向错误的原因
- **典型表现**：错误信息说 A 有问题，但实际是 B 导致的
- **示例**：报 "shape mismatch" 但实际是 dtype 不支持

### C5 - 意外中断

- **优先级**：致命
- **层级**：Session级
- **根因归属**：两者
- **信号描述**：任务在某个点停止，未达到预期目标
- **典型表现**：session 结束时任务未完成，用户表达不满
- **示例**：operator 开发到一半因无法解决的框架问题被迫放弃

## 高优先级（8 个）

严重影响效率或成功率，应优先修复。

### D2 - 文档不完整

- **优先级**：高
- **层级**：实体级
- **根因归属**：文档
- **信号描述**：找到文档但缺少关键信息（参数、返回值、示例）
- **典型表现**：文档存在但信息不全，需要额外搜索或试错
- **示例**：`pypto.Tile` 文档缺少参数类型说明

### D5 - 示例缺失

- **优先级**：高
- **层级**：实体级
- **根因归属**：文档
- **信号描述**：需要代码示例但只有文字描述
- **典型表现**：文档只有 API 签名和文字说明，没有可运行的代码示例
- **示例**：`pypto.scatter` 文档只有参数列表没有使用示例

### E4 - 错误信息无建议

- **优先级**：高
- **层级**：实体级
- **根因归属**：框架
- **信号描述**：错误信息只说"失败"但不说如何修复
- **典型表现**：错误信息缺少修复提示、文档链接、或可能的原因
- **示例**：报 "unsupported dtype" 但不说支持哪些 dtype

### C1 - 反复重试

- **优先级**：高
- **层级**：实体级
- **根因归属**：两者
- **信号描述**：同一操作被重复尝试多次
- **典型表现**：相同或高度相似的工具调用在 session 中反复出现
- **示例**：对 `pypto.Tile` 的编译尝试了 5 次不同的参数组合

### C4 - 过度探索

- **优先级**：高
- **层级**：实体级
- **根因归属**：两者
- **信号描述**：为完成简单任务进行了大量搜索和读取
- **典型表现**：围绕同一实体的 Grep/Glob/Read 操作数量远超正常水平
- **示例**：为了弄清 `pypto.add` 的用法搜索了 15 个文件

### C6 - 用户介入

- **优先级**：高
- **层级**：Session级
- **根因归属**：两者
- **信号描述**：需要用户多次手动干预才能继续
- **典型表现**：用户多次提供修正信息、手动指导操作方向
- **示例**：用户 3 次手动告诉 LLM 正确的 API 用法

### A3 - 参数限制

- **优先级**：高
- **层级**：实体级
- **根因归属**：框架
- **信号描述**：某些合理的参数组合不被支持
- **典型表现**：合理的参数组合报错，需要 workaround
- **示例**：`pypto.matmul` 不支持 float16 输入

### A5 - 边界未定义

- **优先级**：高
- **层级**：实体级
- **根因归属**：框架
- **信号描述**：某些边界条件行为未定义或崩溃
- **典型表现**：特殊输入（空 tensor、极大维度等）导致崩溃
- **示例**：shape 为 [0, 128] 的 tensor 导致 segfault

## 中优先级（2 个）

有一定影响，建议修复。

### A4 - 类型不匹配

- **优先级**：中
- **层级**：实体级
- **根因归属**：框架
- **信号描述**：类型转换失败、shape 不匹配等
- **典型表现**：dtype 或 shape 相关的错误
- **示例**：int32 到 float16 的隐式转换失败

### C2 - 路径摇摆

- **优先级**：中
- **层级**：实体级
- **根因归属**：两者
- **信号描述**：在多个方案间反复切换，无法决策
- **典型表现**：尝试方案 A → 放弃 → 尝试方案 B → 放弃 → 回到方案 A
- **示例**：在 Tile 和 手动循环 之间反复切换实现方式

---

## 层级分类

| 层级 | 断裂点类型 | 说明 |
|------|------------|------|
| **实体级** | D1-D6, A1-A5, E1-E4, C1, C2, C4 | 针对特定实体的问题 |
| **Session级** | C5, C6 | 整个 session 级别的问题 |
