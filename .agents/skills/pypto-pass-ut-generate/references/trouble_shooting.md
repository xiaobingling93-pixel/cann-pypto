## 常见错误速查

### 编译错误

| 错误 | 快速修复 |
|-------|---------|
| `Operations()().()` 语法错误 | 只调用一次 `()`：`Operations().size()` |
| 中文字符混入代码 | 使用英文输入法 |
| 重复定义变量 | 每个测试用例使用不同变量名 |
| 中文冒号 | 使用英文冒号 `:` |

### 逻辑错误

| 错误 | 快速修复 |
|-------|---------|
| 编译阶段配置错误 | 根据 Pass 目录选择：`CS_TENSOR_GRAPH` / `CS_TILE_GRAPH` / `CS_EXECUTE_GRAPH` |
| 期望操作数错误 | 先运行 Pass 打印实际操作数，再调整期望值 |
| 未验证数据类型转换 | 验证 Cast 操作后的数据类型 |
| 未验证 Tensor 连接关系 | 验证消费者和生产者数量 |

### 内存管理

| 错误 | 快速修复 |
|-------|---------|
| 未使用智能指针 | 使用 `std::make_shared` 创建对象 |
| Operation 引用错误 | `AddOperation` 返回引用，用引用接收 |
| 内存泄漏 | 使用智能指针管理所有动态资源 |

### 数据结构

| 错误 | 快速修复 |
|-------|---------|
| Shape 比较错误 | 使用 `GetShape()` 方法 |
| DataType 比较错误 | 使用 `GetDatatype()` 方法 |
| 直接访问成员变量 | 使用 getter 方法访问 |

### Pass 执行

| 错误 | 快速修复 |
|-------|---------|
| 未调用 PreCheck/PostCheck | 调用并验证返回值 |
| Pass 返回值未验证 | 使用 `EXPECT_EQ` 验证返回值 |
| 未验证副作用 | 验证操作类型、数量、连接关系 |

### 遍历相关

| 错误 | 快速修复 |
|-------|---------|
| 遍历时修改容器 | 先复制操作列表：`DuplicatedOpList()` |
| 迭代器失效 | 使用复制的列表遍历 |

## 环境问题处理

### 1. 编译阶段选择

根据 Pass 所在目录选择正确的编译策略：

| Pass 目录 | 编译阶段 |
|----------|---------|
| `tensor_graph_pass` | `CS_TENSOR_GRAPH` |
| `tile_graph_pass` | `CS_TILE_GRAPH` |
| `block_graph_pass` | `CS_EXECUTE_GRAPH` |

### 2. 环境重置

在 `SetUp()` 中确保环境干净：

```cpp
void SetUp() override {
    Program::GetInstance().Reset();
    config::Reset();
    config::SetHostOption(COMPILE_STAGE, CS_EXECUTE_GRAPH);
    // ... 其他配置
}
```

### 3. NPU 架构恢复

测试用例结束前恢复默认架构：

```cpp
// 测试开始
Platform::Instance().GetSoc().SetNPUArch(NPUArch::DAV_2201);
// ... 测试代码
// 测试结束
Platform::Instance().GetSoc().SetNPUArch(NPUArch::DAV_UNKNOWN);
```

## 最佳实践

### 避免错误的最佳实践

1. **仔细检查代码语法**：特别是函数调用和括号匹配
2. **使用不同的变量名**：避免重复定义变量
3. **使用智能指针**：避免内存泄漏
4. **使用 getter 方法**：访问成员变量时使用 `GetShape()`、`GetDatatype()` 等
5. **先复制再修改**：遍历时先复制容器：`DuplicatedOpList()`
6. **验证 Pass 返回值**：使用 `EXPECT_EQ` 验证
7. **验证图结构**：验证操作和 Tensor 的连接关系

### 测试设计原则

1. **覆盖所有功能点**：为每个功能点设计测试用例
2. **测试边界情况**：空图、单个操作、操作链等
3. **验证图结构**：不仅验证操作数，还要验证连接关系
4. **逐步验证**：先验证简单场景，再验证复杂场景
