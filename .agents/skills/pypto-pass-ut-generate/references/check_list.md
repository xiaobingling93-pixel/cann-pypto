# Pass UT 生成检查清单

## 执行前检查

### 1. 环境准备
- [ ] 确认 pypto 仓库路径
- [ ] 确认测试文件路径是否存在 (`pypto/framework/tests/ut/passes/src/`)
- [ ] 确认 pass 对应的实现文件路径

### 2. 业务分析
- [ ] 分析 pass 的核心功能
- [ ] 识别需要测试的关键场景
- [ ] 确定测试用例类型（正向/负向/边界）

### 3. 代码理解
- [ ] 阅读 pass 实现代码
- [ ] 理解数据流和关键逻辑
- [ ] 识别可能的分支路径

## 生成过程检查

### 4. 测试文件创建
- [ ] 使用正确的测试类名 (`Test{pass_name}Pass`)
- [ ] 包含正确的头文件引用
- [ ] 实现 `SetUp()` 和 `TearDown()` 方法

### 5. 测试用例编写
- [ ] 为每个测试用例添加注释说明
- [ ] 使用合适的 shape 和数据类型
- [ ] 正确设置 inCast 和 outCast
- [ ] 验证 pass 执行结果

### 6. 代码规范
- [ ] 避免使用魔鬼数字
- [ ] 使用 const 修饰常量
- [ ] 添加必要的空行和缩进

## 验证检查

### 7. 编译验证
- [ ] 执行 `python3 build_ci.py -c -u=TestPassName.* -j=24 -f=cpp`
- [ ] 确认编译无错误无警告

### 8. 运行验证
- [ ] 执行 UT 测试用例
- [ ] 验证测试通过

### 9. 覆盖率检查
- [ ] 执行 `python3 build_ci.py -c -u=TestPassName.* --gcov -j=24 -f=cpp`
- [ ] 检查覆盖率是否 >= 80%
- [ ] 识别未覆盖的代码行

## 输出检查

### 10. 文档更新
- [ ] 更新 SKILL.md 中的示例
- [ ] 记录覆盖率的改进情况

---

## 常见问题速查

| 问题 | 解决方案 |
|------|----------|
| 编译超时 | 使用 `-u=TestPassName.TestCaseName` 单独运行 |
| 测试失败 | 检查 tensor 的 shape 和 memory type |
| 覆盖率低 | 增加更多场景的测试用例 |
| 断言失败 | 验证 pass 处理后的 expected 值 |
